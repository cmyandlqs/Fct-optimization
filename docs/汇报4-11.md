## 推理速度

尝试多种方案，提升前是数百毫秒(固定学习率迭代200个step)

现在方案是：动态学习率+早停

### 优化算法机制

**优化器**: Adam

**初始学习率**: 20.0（较大起步，快速靠近最优区间）

**学习率调度**: ReduceLROnPlateau
- 监控指标: 损失值 (loss)
- patience = 10：连续 10 步 loss 不下降时触发衰减
- factor = 0.5：每次衰减学习率减半
- min_lr = 0.1：学习率下限
- 衰减序列: 20 → 10 → 5 → 2.5 → 1.25 → 0.625 → 0.1

**早停机制**:
- 监控指标: FCT 实际值（反标准化后的微秒值）
- patience = 20：连续 20 步未刷新历史最优 FCT 则停止
- 记录策略: 始终保留历史最优的 (T, FCT) 作为最终结果，而非最后一步

**最大迭代次数**: 100 步

**整体流程**:

```
lr=20 ──快速靠近最优──→ lr=10 ──精调──→ lr=5 ──微调──→ lr=0.1
  │                      │               │             │
  │  loss 10步不降时     │               │             │
  │  lr × 0.5            │               │             │
  └──────────────────────┴───────────────┴─────────────┘
                         ↑
                    FCT 连续20步无改善 → 早停
```

**效果**: 大部分场景在 40 步通过早停结束，单次推理耗时 50-120ms
