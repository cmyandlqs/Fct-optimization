# 方案设计

## 问题重构

### 原始问题

给定网络流量场景，找到使 FCT（Flow Completion Time，流完成时间）最小的调度阈值 T1 和 T2。

### 重构后的核心洞察

**4 种流量模型是独立的问题，不需要混合训练。**

每个流量模型的特征（avg_flow_size、long_flow_ratio）是固定的，问题天然地按流量类型划分。因此拆分为 4 个独立的子问题，通过**门控路由**选择对应模型处理。

### 门控路由

根据流的平均大小 `avg_flow_size` 判断属于哪种类型：

| 区间 | 流量模型 | 典型场景 |
|------|---------|---------|
| ~64 KB | server | 小流为主 |
| ~630 KB | cache | 中流为主 |
| ~1.6 MB | search | 大流为主 |
| ~7.4 MB | mine | 超大流为主 |

推理时根据 `avg_flow_size` 落在哪个区间，自动路由到对应子模型。

---

## 子问题的数学抽象

以 **cache 模型**为例（其他模型同理）：

- **固定**：T2 = 1024 KB
- **需优化**：T1
- **输入**：load（连续，0.1~0.9）
- **目标**：找到最优 T1 使 FCT 最小

数学形式：

$$FCT = f(T_1, load)$$

给定一个新的 load 值，找到 $T_1^*$ 使得：

$$T_1^* = \arg\min_{T_1 \in [T_{1,min}, T_{1,max}]} f(T_1, load)$$

约束：$T_1 \in [10, 250]$ KB

## 选定方案：回归 + 梯度下降

### 核心思想

分两步走：

1. **训练阶段**：用神经网络学习 $FCT = f(T_1, load)$
2. **推理阶段**：把 T1 当成可学习参数，用梯度下降在连续空间中找最优值

---

### 训练阶段

#### 模型输入输出

对于 cache 模型（优化 T1）：

```
输入 (2维): [T1_kb, load]
输出: log(FCT)
```

对于 search/mine 模型（优化 T2）：

```
输入 (2维): [T2_kb, load]
输出: log(FCT)
```

#### 网络结构

每个子模型独立的小型 MLP：

```
输入 (2维)
  ↓
FC(64) → ReLU → Dropout(0.2)
  ↓
FC(32) → ReLU
  ↓
FC(1) → Linear
  ↓
输出: log(FCT)
```

4 个子模型各自独立训练，互不共享权重。

#### 损失函数

在标准化 log(FCT) 空间计算 MSE：

$$\mathcal{L} = \frac{1}{N} \sum_{i=1}^{N} (\log \widehat{FCT}_i - \log FCT_i)^2$$

#### 训练配置

| 参数 | 值 |
|------|-----|
| 优化器 | Adam (lr=0.001) |
| 正则化 | weight_decay=1e-5, Dropout=0.2 |
| 数据划分 | 70% 训练 / 15% 验证 / 15% 测试 |
| 早停 | patience=30 |

#### 为什么要标准化 log(FCT)

- 不同 load 下 FCT 差异极大（load=0.1 时 ~200ms，load=0.9 时 ~2000ms）
- log 变换压缩尺度，模型更容易学习
- StandardScaler 进一步归一化，加快收敛

---

### 推理阶段

#### 数学推导

目标：给定 load，求最优 T1

$$T_1^* = \arg\min_{T_1} \; f(T_1, \text{load})$$

梯度下降更新公式：

$$T_1^{(k+1)} = T_1^{(k)} - \alpha \cdot \frac{\partial f}{\partial T_1}\bigg|_{T_1^{(k)}, \text{load}}$$

其中 $\alpha$ 是学习率，$\frac{\partial f}{\partial T_1}$ 是 FCT 对 T1 的偏导数。

#### 直观理解

| 梯度符号 | 含义 | 更新方向 |
|---------|------|---------|
| $\frac{\partial f}{\partial T_1} > 0$ | T1 增大 → FCT 增大 | T1 应该减小 |
| $\frac{\partial f}{\partial T_1} < 0$ | T1 增大 → FCT 减小 | T1 应该增大 |

梯度告诉你"T1 往哪个方向调，FCT 会变小"，反复迭代直到收敛。

#### 算法流程

```python
# 初始化可学习的阈值
T = torch.tensor([初始值], requires_grad=True)
optimizer = Adam([T], lr=10.0)

for i in range(max_iter):
    optimizer.zero_grad()

    # 前向传播：预测 FCT
    y_pred = model(T, load)

    # 反向传播：求梯度 ∂FCT/∂T
    y_pred.backward()

    # 更新 T 使 FCT 减小
    optimizer.step()

    # 约束范围
    T.data.clamp_(T_min, T_max)
```

#### 收敛条件

极值点处梯度为零：

$$\frac{\partial f}{\partial T_1} \approx 0 \quad \Longrightarrow \quad T_1^{(k+1)} \approx T_1^{(k)}$$

实际迭代 50~200 次即可收敛。

#### 推理时间

200 次梯度下降迭代在 CPU 上约 **几十毫秒**，完全满足实时需求。

## 方案对比

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **A: 回归 + 梯度下降** | 训 FCT 函数，推理时梯度下降找最优 | 可微、连续最优、泛化好 | 需要训练 4 个模型 |
| B: 直接回归阈值 | 端到端回归最优 T | 推理快（无迭代） | 标签是离散最优，非真正最优 |


**选定方案 A**。主要优势：
- 能在连续空间中找到比离散实验更精确的最优阈值
- 可解释：梯度清晰表达 FCT 随 T 变化的趋势
- 泛化能力强，对未测过的 load 也能推理

## 架构设计

### 子模型命名

每个子模型对应一个流量模型，存储在独立目录：

```
models/
  cache/           # cache 子模型
    predictor.pth  # 模型权重
    scaler.pkl     # 标准化器
  server/         # server 子模型
  search/          # search 子模型
  mine/            # mine 子模型
```

### 门控路由

```python
def route_model(avg_flow_size):
    """根据 avg_flow_size 路由到对应子模型"""
    if avg_flow_size < 200 * 1024:      # < 200KB
        return 'server'
    elif avg_flow_size < 1 * 1024 * 1024:  # < 1MB
        return 'cache'
    elif avg_flow_size < 3 * 1024 * 1024:  # < 3MB
        return 'search'
    else:
        return 'mine'
```

### 统一推理接口

```python
def find_optimal_threshold(avg_flow_size, load):
    """
    给定流量特征和负载，找到最优阈值

    参数:
        avg_flow_size: 平均流大小（字节）
        load: 网络负载 (0-1)

    返回:
        (optimal_threshold, predicted_fct)
    """
    model_name = route_model(avg_flow_size)
    optimizer = ThresholdOptimizer(model_name)
    return optimizer.find_optimal(load)
```

  端到端模型的核心问题

  标签质量有问题：

  你的训练数据是：
  model, load, t1_kb, t2_kb, avg_fct
  cache, 0.1, 100.0, 1024.0, 197227
  cache, 0.1, 10.0, 1024.0, 196999  ← 这个 FCT 最低
  cache, 0.1, 125.0, 1024.0, 197348

  端到端模型会学到：cache + load=0.1 → 最优 T ≈ 10KB

  但问题是：
  - 真实的最优点可能在 T=8KB 或 T=15KB
  - 你永远不知道，因为没测过
  - 你的标签本身就是局部最优，不是全局最优

  这就是"死胡同"的根本原因

  没有"正确答案"，所以没法用监督学习。只能：
  1. 先学 FCT 函数（FCT = f(T, load) 是有"正确答案"的，因为每个数据点都测过了）
  2. 再用优化方法（梯度下降）在学到的函数上找最优
