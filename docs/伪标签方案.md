  确实没有"真实最优 T"作为标签，因为：
  - 仿真实验只测了离散的 T 值
  - 真实的最优点可能在两个离散点之间

  解决方案：生成伪标签

  可以用现有的梯度下降方法生成训练数据：

  当前方法（梯度下降）                          端到端模型
  ────────────────────    ─────────────────►    ──────────────
  model, load  ──► 梯度下降 ──►  optimal_T      model, load  ──►  optimal_T
       (慢，但可以作为"伪标签生成器")
  (快，推理一次前向传播)

  训练数据生成步骤

  # 1. 对每个 (model, load) 用梯度下降求出最优 T
  for model in ['server', 'cache', 'search', 'mine']:
      for load in [0.1, 0.2, 0.3, ..., 0.9]:
          optimal_T = gradient_descent(model, load)  # 当前方法，耗时 400ms
          # 生成训练样本
          X.append([model_embedding, load])
          y.append(optimal_T)

  # 2. 训练端到端模型：input=(model, load) → output=optimal_T
  model = train_regressor(X, y)

  # 3. 推理时
  optimal_T = model.predict([['server', 0.5]])  # < 1ms

  这样训练的模型效果取决于

  1. 梯度下降结果的质量：当前模型 MAPE 高，生成的标签本身就有误差
  2. 覆盖度：可以生成任意 load 的标签，不限于训练数据的 9 个点

  但这是正确的方向吗？

  问题：这样训练的模型只是近似梯度下降，精度不会超过梯度下降本身。

  真正的价值：是速度提升（1ms vs 400ms），而不是精度提升。

  如果要提升精度，根本上是需要更好的 FCT 预测模型（更低的 MAPE），而不是改变推理方式。
