# 核心问题分析：标签生成逻辑的错误

## 🔴 现有方案的致命缺陷

### 当前错误的标签生成逻辑

```python
# 现有代码（preprocess_data.py）
def extract_labels(df):
    # 1. 按场景分组
    # 2. 在测试过的离散阈值中找FCT最小的
    optimal_idx = df.groupby(['model', 'load', 'scenario'])['avg_fct'].idxmin()

    # 3. 将这个离散阈值作为"最优"标签
    labels['t1_kb'] = 最优FCT对应的t1
    labels['t2_kb'] = 最优FCT对应的t2
```

### 问题所在

假设实验测试了这些T1值（固定T2=1024KB）：
```
T1 = [10, 25, 50, 75, 100, 125] KB
FCT = [205, 198, 195, 193, 196, 200] μs
```

现有代码会选 **T1=75KB**（FCT=193μs最小）作为标签。

**但问题是**：
1. ❌ 真正的最优值可能是 **T1=68KB** 或 **T1=82KB**
2. ❌ 你根本没有测试这些值！
3. ❌ 训练模型学习"预测75KB"毫无意义，这只是离散采样的局部最优

---

## 🎯 问题的本质

### 这不是标准回归问题！

**标准回归**（现有错误做法）：
```
输入: [load=0.5, flow_count=10000, ...]
输出: T1=75 (从离散采样中选的"最优")
损失: MSE(T1_pred, 75)
```

**实际问题**：
```
输入: [load=0.5, flow_count=10000, ...]
目标: 找到连续空间中的 T*，使 FCT(T*) 最小
约束: T* ∈ [10, 250] KB
```

这是一个**优化问题**，不是简单的回归问题！

---

## ✅ 正确的解决方案

### 方案对比

| 方案 | 描述 | 优势 | 劣势 | 推荐度 |
|------|------|------|------|--------|
| **方案1：FCT预测函数** | 训练模型预测 FCT=f(阈值,特征)，然后用优化找最优阈值 | 理论正确，可学到FCT规律 | 需要额外优化步骤 | ⭐⭐⭐⭐⭐ |
| **方案2：强化学习** | State=特征, Action=阈值, Reward=-FCT | 直接优化目标 | 数据需求大，训练不稳定 | ⭐⭐⭐ |
| **方案3：改进回归** | 密集采样 + 插值平滑标签 | 实现简单 | 仍受采样密度限制 | ⭐⭐⭐⭐ |
| **方案4：两阶段模型** | 先预测FCT函数，再优化 | 结合1和3的优势 | 实现复杂 | ⭐⭐⭐⭐ |

## 🏆 推荐方案：FCT预测函数 + 优化

### 核心思想

**不预测阈值，预测FCT函数！**

```
训练阶段:
  模型学习: FCT = f(T1, T2, load, flow_count, ...)

推理阶段:
  给定场景特征 → 用优化算法找 (T1*, T2*)
  使 FCT(T1*, T2*, features) 最小
```

### 具体实现

#### 1. 训练FCT预测模型

```python
# 输入: [t1_kb, t2_kb, load, flow_count, avg_flow_size, long_flow_ratio, ...]
# 输出: avg_fct
# 损失: MSE(FCT_pred, FCT_true)

class FCTPredictor(nn.Module):
    def __init__(self, input_dim=8, hidden_dims=[128, 64, 32]):
        super().__init__()
        # 输入包含阈值 + 场景特征
        self.network = nn.Sequential(
            nn.Linear(input_dim, hidden_dims[0]),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden_dims[0], hidden_dims[1]),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(hidden_dims[1], hidden_dims[2]),
            nn.ReLU(),
            nn.Linear(hidden_dims[2], 1)  # 输出FCT
        )

    def forward(self, x):
        return self.network(x)
```

#### 2. 推理时优化阈值

```python
def find_optimal_threshold(model, features):
    """
    使用训练好的FCT预测模型，找最优阈值

    参数:
        model: FCT预测模型
        features: 场景特征 [load, flow_count, ...]

    返回:
        (T1_opt, T2_opt): 最优阈值
    """
    import torch.optim as optim

    # 1. 初始化可学习的阈值参数
    T1 = torch.tensor([50.0], requires_grad=True)  # 初始值
    T2 = torch.tensor([1024.0], requires_grad=True)

    # 2. 优化器
    optimizer = optim.Adam([T1, T2], lr=10.0)

    # 3. 梯度下降优化
    for i in range(100):
        optimizer.zero_grad()

        # 构造输入
        x = torch.cat([
            T1,  # 可学习的T1
            T2,  # 可学习的T2
            torch.tensor(features)  # 固定的场景特征
        ])

        # 预测FCT
        fct_pred = model(x)

        # 最小化FCT
        loss = fct_pred

        # 反向传播（更新T1, T2）
        loss.backward()
        optimizer.step()

        # 约束在合理范围内
        T1.data.clamp_(10, 250)
        T2.data.clamp_(500, 2048)

    return T1.item(), T2.item()
```

### 优势分析

| 优势 | 说明 |
|------|------|
| ✅ **理论正确** | 直接优化目标函数FCT |
| ✅ **连续优化** | 不受离散采样限制 |
| ✅ **利用所有数据** | 每条实验记录都有用（不只是FCT最小的） |
| ✅ **可解释性** | 可以看到FCT随阈值变化的曲线 |
| ✅ **泛化能力** | 学到的是FCT规律，不是特定阈值 |

## 🔧 实现细节

### 特征工程

```python
def prepare_fct_training_data(df_raw):
    """
    准备FCT预测模型的训练数据

    输入: 原始实验数据（每条是一个阈值组合的测试结果）
    输出: (X, y) 训练数据
    """
    X = []
    y = []

    for _, row in df_raw.iterrows():
        # 输入特征: [t1, t2, load, flow_count, avg_flow_size, long_flow_ratio, ...]
        features = [
            row['t1_kb'],
            row['t2_kb'],
            row['load'],
            row['flow_count'],
            MODEL_CONFIG[row['model']]['avg_flow_size'],
            MODEL_CONFIG[row['model']]['long_flow_ratio'],
            # ... 其他特征
        ]

        # 输出: avg_fct
        target = row['avg_fct']

        X.append(features)
        y.append(target)

    return np.array(X), np.array(y)
```
